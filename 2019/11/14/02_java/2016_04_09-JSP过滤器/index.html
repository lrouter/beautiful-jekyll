过滤器

JSP过滤器，用于：

  • 在客户端的请求访问后端资源之前，拦截这些请求。
  • 在服务器的响应发送回客户端之前，处理这些响应。

JSP过滤器，有多种类型：

  • 身份验证过滤器（Authentication Filters）。
  • 数据压缩过滤器（Data compression Filters）。
  • 加密过滤器（Encryption Filters）。
  • 触发资源访问事件过滤器。
  • 图像转换过滤器（Image Conversion Filters）。
  • 日志记录和审核过滤器（Logging and Auditing Filters）。
  • MIME-TYPE 链过滤器（MIME-TYPE Chain Filters）。
  • 标记化过滤器（Tokenizing Filters）。
  • XSL/T 过滤器（XSL/T Filters），转换 XML 内容。

过滤器的使用方式

过滤器是一个接一个的串在一起的，例如用户的请求到来时，会按照顺序一个个的经过这些过滤器。

考虑1个过滤器的场景。

要使用过滤器，就需要知道当用户访问什么资源时经过该过滤器，以及过滤器对应的处理java代码（继承Filter JAVA类）。

例如，我们要创建1个LogFilter过滤器，在用户访问网站上的所有资源时经过该过滤器。该过滤器对应的JAVA代码是LogFilter类。

过滤器处理类继承自Filter类，可以New->Filter来创建这样的类。

该类有3个基本方法：

  • public void init(FilterConfig filterConfig), 初始化方法。JSP容器启动时，创建该Filter类的实例，调用该方法。该方法接收web.xml文件中通过init-param定义的初始化参数。
  • public void doFilter (ServletRequest, ServletResponse, FilterChain), 过滤动作所对应的方法。
  • public void destroy(), Servlet容器在销毁过滤器实例前调用该方法，在该方法中释放Servlet过滤器占用的资源。
