HTTP响应头

1个服务器回复的HTTP响应报文，包含以下头部信息：

响应头 描述
Allow 指定服务器支持的request方法（GET，POST等等）
Cache-Control 指定响应文档能够被安全缓存的情况。通常取值为 public，private 或no-cache 等等。 Public意味着文档可缓存，Private意味着文档只为单用户服务并且只能使用私有缓存。No-cache 意味着文档不被缓存。
Connection 命令浏览器是否要使用持久的HTTP连接。close值 命令浏览器不使用持久HTTP连接，而keep-alive 意味着使用持久化连接。
Content-Disposition 让浏览器要求用户将响应以给定的名称存储在磁盘中
Content-Encoding 指定传输时页面的编码规则
Content-Language 表述文档所使用的语言，比如en， en-us,，ru等等
Content-Length 表明响应的字节数。只有在浏览器使用持久化 (keep-alive) HTTP 连接时才有用
Content-Type 表明文档使用的MIME类型
Expires 指明啥时候过期并从缓存中移除
Last-Modified 指明文档最后修改时间。客户端可以 缓存文档并且在后续的请求中提供一个 If-Modified-Since请求头
Location 在300秒内，包含所有的有一个状态码的响应地址，浏览器会自动重连然后检索新文档
Refresh 指明浏览器每隔多久请求更新一次页面。
Retry-After 与503 (Service Unavailable)一起使用来告诉用户多久后请求将会得到响应
Set-Cookie 指明当前页面对应的cookie

JSP Responce对象

JSP的Responce对象，提供了一系列的方法，用于操作HTTP响应头部。

可以添加新的cookie或时间戳，还有HTTP状态码等等。

例如：

  • void addCookie(Cookie cookie) 添加指定的cookie至响应中
  • void addIntHeader(String name, int value) 添加指定名称的响应头和int值

HTTP头部信息

客户端浏览器发出的HTTP请求，会带有一个HTTP头部，该头部包含以下信息，这些信息贯穿了JAVA WEB编程的所有项目：

信息 描述
Accept 指定浏览器或其他客户端可以处理的MIME类型。它的值通常为 image/png 或 image/jpeg
Accept-Charset 指定浏览器要使用的字符集。比如 ISO-8859-1
Accept-Encoding 指定编码类型。它的值通常为 gzip 或compress
Accept-Language 指定客户端首选语言，servlet会优先返回以当前语言构成的结果集，如果servlet支持这种语言的话。比如 en，en-us，ru等等
Authorization 在访问受密码保护的网页时识别不同的用户
Connection 表明客户端是否可以处理HTTP持久连接。持久连接允许客户端或浏览器在一个请求中获取多个文件。Keep-Alive 表示启用持久连接
Content-Length 仅适用于POST请求，表示 POST 数据的字节数
Cookie 返回先前发送给浏览器的cookies至服务器
Host 指出原始URL中的主机名和端口号
If-Modified-Since 表明只有当网页在指定的日期被修改后客户端才需要这个网页。 服务器发送304码给客户端，表示没有更新的资源
If-Unmodified-Since 与If-Modified-Since相反， 只有文档在指定日期后仍未被修改过，操作才会成功
Referer 标志着所引用页面的URL。比如，如果你在页面1，然后点了个链接至页面2，那么页面1的URL就会包含在浏览器请求页面2的信息头中
User-Agent 用来区分不同浏览器或客户端发送的请求，并对不同类型的浏览器返回不同的内容

JSP Request对象

Request对象提供了一系列的方法，用来处理客户端请求的HTTP头部信息。包括表单数据，cookies，HTTP方法等等。

例如：

  • String getRemoteHost() 返回客户端的完整名称
  • HttpSession getSession() 返回request对应的session对象，如果没有，则创建一个
  • boolean isSecure() 返回request是否使用了加密通道，比如HTTPS

JSP概述

什么是JSP?

down vote
accepted
Short answer:

JSP is runs on the server whereas javascript runs on the client.

Long answer:

JSP (JavaServer Pages) is a Java technology for creating dynamic web pages which is deployed by a servlet container. Both are run server-side and require a webhost of some kind (e.g., Tomcat). At runtime, jsp files will be compiled into servlets and executed when requested. It is comparable to languages such as PHP or ASP. JSP can be described as a technology for changing the contents of a page.
Javascript is a scripting language that is run on the client. Its use is primarily for updating the DOM or dynamically fetching/sending data. Javascript can be described as a technology which changes the behavior or presentation of a page.

JSP拥有JAVA语言的所有特性：
面向对象；
多线程；
健壮;

JSP运行JSP容器中：Tomcat，Jetty

JSP -> Serverlet -> Class文件

JSP历史

URN

Uniform Resource Name，代表某个资源独一无二

URI

超文本传输协议

发送和接收HTML页面

特点：

  1. 支持客户和服务器模式
  2. 无连接
  3. 无状态，对于事务处理的结果没有记忆

方法：
GET: 向特定的资源发送请求
POST：向特定的服务区发送资源

JDBC

JDBC就是一项服务，专门针对关系型数据库。

用户程序

JAVA

JDBC（规定了数据连接和操作的接口）

ORACLE MYSQL
（各种数据库实现了接口）

JDBC连接数据的两种方式

  • JDBC-ODBC
  • JDBC连接，程序 -> JDBC -> 数据库

JDBC网络连接：

JDBC的所有开发操作都在java.sql包里面。

连接数据库

Connection接口:

public static Connection getConnection(String url,Properties info) throws SQLException

工厂类屏蔽了各种数据库的驱动，用户程序只要接口和工厂类。

操作数据库

###Statement接口:

Statement createStatement() throws SQLException

数据库的操作主要有两类：

更新数据库：int executeUpdate(String sql) throws SQLException，返回更新行数；

查询数据库：ResultSet executeQuery(String sql) throws SQLException，

传入sql语句就可以直接返回数据库的结果。对于数据库的返回结果ResultSet，可以使用java语言进行操作。

不推荐使用Statement接口

批处理

JDBC支持对数据库的批处理操作，也就是一次支持多条sql语句的执行。

###PreparedStatement接口
PreparedStatement prepareStatement(String sql) throws SQLException

数据库的操作主要有两类：

更新数据库：int executeUpdate() throws SQLException；

查询数据库：ResultSet executeQuery() throws SQLException；

sql语句的值，不再硬编码了。在先使用占位符“?” 一个sql语句，基于sql创建PreparedStatement之后，使用setxxx子类的代码设置sql，最后再调用上面的
数据库操作函数。

事务处理

事务处理，就是批处理里面包含的所有sql语句，要么一起c成功执行，要么一起失败。

  • 设置是否自动提交 setAutoCommit
  • 设置提交事务 commit
  • 回滚 rollback
